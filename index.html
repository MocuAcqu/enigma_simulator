恩尼格瑪機的歷史

恩尼格瑪密碼機（Enigma）最初由德國工程師阿瑟·謝爾比烏斯於一戰末期發明，原本目的是用於商業機密通訊。然而在1920年代，德國軍方看中了其複雜的加密能力，將其改裝並廣泛應用於二戰期間的陸、海、空軍通訊。

德國人深信恩尼格瑪機是不可破譯的，因為其可能的設定組合高達 158,962,555,217,826,360,000 種。然而，波蘭密碼局率先發現了其運作規律，並將研究成果交給了英國。

在英國布萊切利園（Bletchley Park），以艾倫·圖靈（Alan Turing）為首的密碼學家們研發出了「炸彈機 (Bombe)」，成功自動化了破譯過程。這項成就據信將二戰縮短了兩年，拯救了數百萬人的生命。

恩尼格瑪機運作原理

恩尼格瑪機是一種「機電轉子密碼機」，它的加密過程是一條複雜的電流迴路：

  1. 鍵盤： 按下一個字母鍵（如 'A'），接通電流。
  2. 接線板： 電流首先通過接線板。如果 'A' 和 'Z' 連接，電流會變成 'Z' 進入機器。
  3. 轉子（核心）： 電流穿過三個轉子。每個轉子內部都有錯綜複雜的交叉接線。最重要的是，每按一次鍵，最右邊的轉子就會轉動一格。這意味著即使連續按兩次 'A'，也會被加密成完全不同的字母（多表密碼）。
  4. 反射器： 電流到達最左邊後，反射器將電流折返回轉子。這設計確保了 加密和解密是同一個過程（如果 A 加密成 K，在相同設定下 K 就會解密成 A），但也導致一個致命弱點：一個字母永遠不會被加密成它自己。
  5. 反向傳輸與顯示： 電流反向穿過轉子，再次經過接線板，最後點亮顯示燈泡。

程式碼機制

恩尼格瑪機的本質是一個複雜的機電迴路。當你按下一個按鍵，電流會跑過一條極度曲折的路徑。以下是本模擬器中，還原歷史硬體邏輯的 JavaScript 核心程式碼：

enigma_core.js

processLetter(char) {
    // 【步驟 1：機械步進】每按一鍵，最右側轉子轉動一格。
    // 遇到特定缺口 (Notch) 時，會觸發「雙步進」帶動左側轉子。
    this.stepRotors(); 

    // 【步驟 2：接線板 (Steckerbrett)】
    // 檢查字母是否在接線板上有互換 (例如 A 換成 Z)
    let current = plugboardMap[char] || char;

    // 【步驟 3：正向穿過轉子 (右 -> 中 -> 左)】
    // 電流根據轉子的內部佈線與當前旋轉角度，改變字母
    for (let i = 2; i >= 0; i--) {
        current = this.rotorPass(current, rotors[i], true);
    }

    // 【步驟 4：反射器 (Umkehrwalze)】
    // 電流折返。這保證了加密與解密路徑相同，但也導致致命弱點：
    // 「一個字母永遠不會被加密成自己」。
    current = this.REFLECTOR[current];

    // 【步驟 5：反向穿過轉子 (左 -> 中 -> 右)】
    for (let i = 0; i <= 2; i++) {
        current = this.rotorPass(current, rotors[i], false);
    }

    // 【步驟 6：再次經過接線板，點亮燈泡】
    return plugboardMap[current] || current;
}

2. 密碼學協定：第二層加密 (The Cryptographic Protocol)

硬體的複雜度只是恩尼格瑪機的一半，另一半的安全來源於德軍嚴格的「操作協定」。如果所有通訊兵一整天都用同一個設定發報，密碼很容易被盟軍透過頻率分析破解。因此，德軍採用了雙重密鑰系統。

第一層：每日密鑰 (Tagesschlüssel)

由最高統帥部分發的密碼本（如底部圖片所示）。全軍每天必須統一設定好轉子編號、環設定、接線板。這是機器的「基礎狀態」。

第二層：訊息密鑰 (Spruchschlüssel)

這是為了每封獨立電報隨機產生的「一次性密碼 (Session Key)」，操作流程如下：

  1. 發報員隨便想一個起點 (例如 W E R) 並轉動轉子到此位置。
  2. 他再隨機想一個這封電報真正的密鑰 (例如 A S D)。
  3. 他在鍵盤上打出 A S D，機器將其加密 (假設亮起 Z X C)。
  4. 發報員將轉子轉到真正的密鑰 A S D，開始打機密內文。
  5. 最後發送的電碼開頭會是明文的起點與加密後的密鑰：WER ZXC [密文...]。

💡 歷史的破綻：在戰爭初期，德軍為了防錯，規定這個密鑰必須「打兩次」(例如輸入 ASD ASD)。正是這個重複的規律，讓波蘭數學家馬里安·雷耶夫斯基 (Marian Rejewski) 找到了破綻，成功推導出早期的機器接線！
